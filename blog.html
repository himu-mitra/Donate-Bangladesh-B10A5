Important FAQ ...

what is the document object model (DOM) ?

The Document Object Model (DOM) is like a map of a webpage that shows how everything is connected. It turns parts of the page, like text, images, and links, into objects that can be easily changed. By using JavaScript, developers can update or modify these parts, making the webpage interactive. For example, they can change colors, add new text, or make things move, all without reloading the whole page.

How do you select an element from the DOM ?

To select an element from the DOM in JavaScript, you can use different methods. `document.getElementById()` selects an element by its ID, which is quick but only works for one element. `document.getElementsByClassName()` picks elements by class, and `document.getElementsByTagName()` selects elements by their tag. `document.querySelector()` is flexible, selecting one element using CSS selectors, while `document.querySelectorAll()` selects multiple elements, but may be slower with large pages. Each method has its own pros and cons depending on the task.

What is the event delegation in the context of the DOM, and why is it useful ?

Event delegation is a way to handle events in a simple and efficient way. Instead of putting event listeners on every element, you place one listener on a parent element. The parent will catch events from its child elements. This helps reduce the number of listeners, saves memory, and works well when new elements are added to the page later.

How do you manipulate an element's attributes and styles using the DOM ?

To change an element's attributes and styles using the DOM, you can use JavaScript. For example, you can use `element.setAttribute()` to change things like an image's `src` or a link's `href`. To change styles, use `element.style` and update properties like color or size. For example, `element.style.color = "red"` makes text red. This allows you to easily change how elements look on a webpage.
